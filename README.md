# Jogos-do-Girotto
Esse README é uma explicação do main.c a qual reside nesse arquivo. Esse código é nada mais que um menu, onde nele acessamos três jogos, sendo eles o de Perguntas e Repostas, o Cobra na Caixa e o Gousmar War. Eu, João Gabriel, junto com os membros do meu grupo, Pietra e André Barbosa, criamos esse código.

Quando criado um código, inicialmente usamos apenas as bibliotecas básicas da linguagem C stdio.h e stdlib.h. A partir disso, criamos um int main com um void, para criar um ponteiro que armazenasse o código. Daí declarando as varíaveis, entramos no ponto principal, o primeiro do while, responsável pela criação inicial do menu, nele contém a opção de entrar no menu principal que tem os jogos, ou a opção de sair, e pra representar eles, temos dois if, no primeiro entra no menu, e no segundo, imprime a mensagem de SAIR DO JOGO, a qual, por quebrar a condição que há no do while, acaba por sair de fato do menu, encerrando o programa. Voltando pro menu de jogos, nele há as opções dos jogos, em que os 3 estão armazenados dentro de um comando switch case, que cria uma estrutura sólida para a facilitação da criação de um menu com várias opções, dentre desse comando, há um total de 3 cases, cada uma delas armazenando um código.

Na primeira Case, a qual é armazenado o jogo Perguntas e Respostas, sua estrutura começa com um do while, a qual é importante para uma condição explicado mais pra frente, e já começando o jogo, ele basicamente é baseado em comandos simples como o printf, o scanf, o if e o else, os if armazenam as respostas corretas, na qual tem uma linha focado em incrimentar com base nos seus acertos, vendo seu resultado final, e os else armazenam o erro, para caso você erre, imprimir a resposta correta. Repetindo isso para cada uma das 5 perguntas, no final, imprima o resultado final, com base no incremento, de quantas perguntas você respondeu corretamente. Depois disso, existe uma opção para repetição do jogo ou saída dele, a partir disso, a questão acaba sendo retomada com relação ao do while, já que a condição será baseada na escolha de repetir ou de sair. Depois disso, fora do laço de repetição, o comando break estará lá para quebrar o código quando desejar sair do jogo.

Na segunda case, armazenando o jogo Cobra na Caixa, já se vê uma estrutura mais complexa, já que nele, primeiramente precisará de uma biblioteca time.h, a explicação entrará de certa forma mais a frente. Semelhante à case 1, nessa case também começaremos com um Do While, a complexidade desse jogo está relacionado ao comando rand, responsável por criar uma aleatoriedade, o problema é que a aleatoridade do rand funciona apenas uma vez, e aí que a biblioteca do time.h entra, já que ele é responsável por reforçar o rand, tornando os números de fato aleátorios, invés da aleatoridade ser sorteada apenas uma vez. Bom, com isso, você criará um rand para 5 pontos, e criará todo um sistema pra as caixas vazias, pro botão e para cobra, o jogador que encontra o botão vence, e o outro perde, ou se encontrar a cobra, o outro ganha. Depois disso, do mesmo jeito que o outro jogo, tem a condição do laço de repetição, que serve pra repetir o jogo ou sair, e o break para quebrar a case caso saia do laço de repetição.

Na terceira e última case, há o jogo Gousmas War, que dentre os três, é o que se mostra mais complexo dos jogos. O funcionamento do jogo é baseado em quatro monstros, a quais chamamos de Gousmas, dois deles com um jogador, e os outros dois com o outro, cada um deles começa com um de fúria, em que, ao ultrapassar cinco, a Gousmas morre, e para passar fúrias, é necessário que o jogador ataque, a quantidade de Gousmas que existe na Gousmas que atacou, será passado para a Gousmas atacada, por exemplo, se eu atacar uma Gousmas com um de fúria, a qual a minha Gousmas tem dois de fúria, a atacada ficará com três. Além disso, também a situação de divisão, onde você pode transferir a fúria de uma Gousmas sua para a outra, com tanto que não zere as fúrias da outra. Esse jogo acabo usando uma biblioteca stdbool.h, utilizado para facilitar a condição de um while, já que neste jogo, é utilizado um while dentro do outro laço derepetição, com uma condição de verdade, e com um finalizador break, para que não repita infinitamente. Com isso, o este jogo teve como base principal os arrays e strings, que serviram principalmente para armazenar informações relacionadas as quantidades de fúrias dos jogadores, e armazenar de maneira certa qual jogador está com certa quantidade de fúrias em determinado ataque ou transferência da mesma.
